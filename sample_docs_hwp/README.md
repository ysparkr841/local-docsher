# HWP/HWPX sample document set

LDS-017 prepares a provenance-safe minimum sample set for HWP/HWPX work.

## Committed synthetic samples

These files are generated specifically for Local Docsher and contain only synthetic text:

- `simple.hwpx` — simple paragraph-only HWPX-like ZIP/XML fixture containing `문서셔HWPX단순토큰`.
- `table.hwpx` — simple HWPX-like ZIP/XML fixture with table-oriented text tokens containing `문서셔HWPX표토큰`.
- `simple.hwp.manifest.md` — manifest for a future or user-provided redistributable HWP5 binary sample containing `문서셔HWP단순토큰`.

The HWPX fixtures are intentionally small ZIP/XML documents so LDS-018 can validate the stdlib HWPX extraction baseline without relying on proprietary tools or unclear public binaries.

## Why `simple.hwp` is not committed yet

Binary `.hwp` sample redistribution rights are easy to get wrong. LDS-017 therefore records the required HWP sample shape in `simple.hwp.manifest.md` instead of committing a possibly non-redistributable binary. A real `simple.hwp` may be added later only after provenance, license/redistribution permission, generator tool/version, and checksum are documented.

## Required Korean search tokens

- `simple.hwpx`: `문서셔HWPX단순토큰`
- `table.hwpx`: `문서셔HWPX표토큰`
- future `simple.hwp`: `문서셔HWP단순토큰`

## Provenance and license

The committed `.hwpx` fixtures were generated by a Local Docsher test-preparation script using Python stdlib `zipfile` and contain only the synthetic Korean tokens above. They are safe to redistribute with this repository.

Do not add HWP/HWPX samples with unclear redistribution rights. Prefer freshly generated documents containing only synthetic text.
